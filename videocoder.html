Videocoder

VideoCoder is a college project with a team of 4 for the module Advanced Rich Internet Applications.

It incorporates the YouTube API to filter relevant 'Learn to Code' Channels.

Technology used:

HTML5, CSS3, AdobeCS
AngularJS
YouTube API
JavaScript
JQuery
JSON
Bootstrap
plugins for chart and login functionality

The code

Please see below for insight on the development along with Wireframes.

Development

The project was developed by an agile team of four using Github, in under three weeks. It explores incorporating an API into an application. The team focused on the increasingly sophisticated field of online learning.

The team wanted a portal to ‘learn to code’ videos that would provide ‘at a glance’ data for the learner, making an efficient learning environment.

For many who have an interest in learning how to code it may be difficult to settle on a good starting point. A popular way to enhance learning and make progress is to watch a video where the learner can follow along at their own pace.

YouTube consists of many thousands of channels catering to these needs. While the quantity may be very encouraging to the novice, the quality of delivery, explanation and results can vary considerably.

VideoCoder was designed to filter the most successful learning videos directly from YouTube to the learner. Who can quickly find the best content for their learning.

As well as filtering content from the YouTube API, the team incorporated a pie chart which integrates seamlessly to display the total amount of views of the top ten videos in the channel. A user can easily gauge which are most popular.

AngularJS is used for the drop down Menus. It lets us take JSON from the API and render it on a local device. It is enabling content on the home page be replaced with content from the selected link with no page reloads.

Design

YouTube displays further related videos to the right, but the VideoCoder design allows the learner to focus primarily on the recommended channel only.

Layout went through some alteration as the team experimented with placement of the ‘beginners’ channels underneath and to the right of the video screen while still having the other channels from the main page available as a dropdown. Feedback from testing these iterations, led having a seperate ‘beginners’ section so the user only has one place to think of for changing channels.

The team knew from personal experience it was important to practice code on the fly, and the JS fiddle tool for front end testing was incorporated. However, from a usability perspective this was removed from smaller screens.

The interface is designed so that the learner will not be distracted. Navigation is available at all times for all of the channels and for the other core functionalities.

Wireframes

The app is responsive for mobile. The team used Bootstrap’s media queries to ensure correct display of videos and charts. With responsive CSS a viewport below 768px will force the columns and rows to become fluid and stack vertically.
